Method Path Auth Description

Quickstart

Create your first refund in under a minute.

1

Authenticate with your API key

Add the X-API-Key header to every mutating request.

# All POST requests require this header
X-API-Key: SOLARA-SECRET-2026
2

Create a full refund

curl -X POST https://refund-service.vercel.app/api/v1/refunds \
  -H "X-API-Key: SOLARA-SECRET-2026" \
  -H "Content-Type: application/json" \
  -d '{
    "transaction_id": "TXN-REG-001",
    "operator_id": "op1",
    "reason": "customer request"
  }'
3

Inspect the response

{
  "data": {
    "refund_id": "RF-F5C0DBC7",
    "status": "APPROVED",
    "total_refund_amount": "18.80",
    "currency": "BRL"
  }
}

Authentication

The API uses a static API key passed via the X-API-Key header.

Security Notice

Never pass the API key in the URL or request body. The key is checked using hmac.compare_digest to prevent timing attacks. After 5 failed attempts, the IP is rate-limited for 60 seconds.

401

Missing or invalid key

429

Rate limited after 5 fails

64KB

Max request body size
